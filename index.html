Intro

왜 데이터를 가공해야 하는가

유저 테스트가 끝나면 수치, 클릭 경로, 응답 텍스트가 쏟아진다.
이 상태로는 아무도 읽지 않는다. 가공은 데이터를 읽히는 것으로 만드는 과정이다.

Raw Data란?
DEFINITION

유저 테스트 직후 수집된 가공되지 않은 데이터.
숫자, 클릭 경로, 인터뷰 발언, 설문 응답 등
아직 의미가 부여되지 않은 원자재다.

WHY IT'S NOT ENOUGH

Raw Data는 "무슨 일이 있었는가"는 알려주지만
"무엇이 문제이고 어떻게 고쳐야 하는가"는
알려주지 않는다. 가공이 그 다리를 만든다.

Chapter 01

데이터의 종류: 4가지 기준

모든 유저 리서치 데이터는 두 가지 축으로 분류된다. 정량 vs 정성, 행동 vs 태도.
어떤 방법론을 쓸지는 "지금 내게 어떤 종류의 질문이 있는가"에 따라 결정된다.

QUANTITATIVE vs QUALITATIVE
정량은 "얼마나, 몇 명이"를 알려주고
정성은 "왜, 어떻게"를 알려준다.
둘 다 있어야 완전한 그림이 된다.
BEHAVIORAL vs ATTITUDINAL
행동은 사용자가 실제로 한 것이고
태도는 사용자가 말한 것이다.
이 둘은 종종 일치하지 않는다.
⚡ KEY INSIGHT
태도와 행동이 불일치하는 지점이
가장 강력한 인사이트를 품고 있다.
예: 성공률 85% + 만족도 2.6점
→ "찾긴 했지만 경험이 나빴다"

Overview

4가지 방법 한눈에 보기

같은 데이터도 어떤 방법으로 보느냐에 따라 전혀 다른 것이 보인다.
4가지 방법은 서로 다른 질문에 답하며, 함께 쓸수록 더 입체적인 그림이 나온다.

01
Affinity
Mapping

인터뷰나 관찰에서 얻은 발언과 행동들을 비슷한 것끼리 묶어 패턴을 찾는 방법. 흩어진 조각들이 하나의 주제로 모일 때, 숫자로는 보이지 않던 문제가 드러난다.

정성 태도
답하는 질문 "사용자들이 공통으로 겪는 불편함의 패턴은 무엇인가?"
02
Metrics
Matrix

테스트에서 나온 수치들을 한 눈에 비교할 수 있는 표로 정리하고, 어떤 문제가 가장 시급한지 우선순위를 매기는 방법. 숫자의 홍수에서 핵심을 건져낸다.

정량 행동
답하는 질문 "어떤 화면에서, 얼마나 많은 사람이 어려움을 겪었는가?"
03
Behavior
Mapping

사용자가 화면을 넘어가는 경로 전체를 흐름도로 시각화하는 방법. 어디서 헤매고, 어디서 포기했는지가 한눈에 보인다. Maze의 Navigation Overview가 바로 이것이다.

정량 행동
답하는 질문 "사용자들은 어떤 경로로 이동했고, 어디서 막혔는가?"
04
Behavioral +
Attitudinal
Correlation

실제 행동 수치와 사용자의 만족도를 나란히 놓고 비교하는 방법. 둘이 일치하지 않는 지점에서 가장 중요한 문제가 숨어 있다.

행동 태도
답하는 질문 "숫자는 괜찮은데 왜 사용자들은 불만족스러워 할까?"

Method 01

Affinity
Mapping

비슷한 것끼리 묶어 패턴 찾기

인터뷰나 관찰 세션에서 얻은 발언들을 포스트잇처럼 하나씩 적고, 비슷한 내용끼리 묶어 그룹을 만드는 방법이다. 개별 발언 하나하나는 의미가 작아 보여도, 묶이고 나면 사용자들이 공통으로 겪는 문제가 선명하게 드러난다.

이럴 때 사용해요
인터뷰나 관찰 메모가 많이 쌓였는데 어디서부터 시작해야 할지 모를 때.
숫자 데이터만으로는 "왜 그랬는지"가 보이지 않을 때.
팀원들과 함께 데이터를 소화하고 싶을 때.
정성 데이터 태도 지향
진행 순서
1
관찰 내용을 낱장으로 기록한다
인터뷰 발언, 관찰한 행동 하나하나를 포스트잇 한 장에 하나씩 적는다. 해석하지 말고 있는 그대로 적는 것이 중요하다.
"설문 버튼이 어디 있는지 한참 찾았어요"
2
비슷한 것끼리 옆에 놓는다
같은 문제를 가리키는 것 같은 메모들을 모아 그룹을 만든다. 처음에는 어색해도 괜찮다. 여러 번 다시 배치하면서 맞는 위치를 찾아간다.
"찾기 어렵다"는 메모들이 한 곳으로 모인다
3
각 그룹에 이름을 붙인다
그룹을 대표하는 주제명을 붙인다. 이것이 하나의 클러스터(Cluster)가 된다. 클러스터 이름이 곧 사용자 문제의 핵심 주제다.
클러스터명: "탐색 어려움"
4
클러스터에서 인사이트를 뽑는다
각 클러스터가 말하는 것이 무엇인지 한 문장으로 정리한다. 이것이 이후 디자인 개선의 출발점이 된다.
"메인 화면에서 주요 기능을 찾기 어렵다"
💡 TIP
디지털 도구를 쓸 때는 FigJam, Miro, Notion을 활용할 수 있다. 포스트잇 색깔로 참여자나 주제를 구분하면 패턴이 더 빨리 보인다.

Method 02 · Example

Metrics Matrix — 실제로 해보면

카카오뱅크 Maze 테스트 결과를 표로 재구성한 예시. Before와 After를 나란히 놓으면 개선 효과가 바로 보인다.

// BEFORE — 기존 디자인 (n=7)
미션 ID 태스크 내용 성공률 Misclick Drop-off 소요 시간 심각도 읽는 법
M-01 '돈 버는 서베이' 기능을 홈 화면에서 찾아보세요 85.7% 79.6% 14.3% 15.1s 중* 성공률은 높지만 Misclick이 비정상적으로 높다. 숨겨진 마찰 존재.
M-02 화면에 보이는 설문지 중 하나를 골라 참여를 완료해주세요 57.1% 65.0% 42.9% 23.4s 상 7명 중 3명 실패. 즉시 개선이 필요한 최우선 대상.
// AFTER — 개선 디자인 (n=4~7)
미션 ID 태스크 내용 성공률 Misclick Drop-off 소요 시간 심각도 변화
M-01 '돈 버는 서베이' 기능을 홈 화면에서 찾아보세요 100% 51.7% 0% 31.3s ⚠ 하 성공률·Drop-off 개선. 단, 소요 시간 증가는 추가 확인 필요.
M-02 화면에 보이는 설문지 중 하나를 골라 참여를 완료해주세요 100% 41.2% 0% 17.3s 하 모든 지표 개선. 심각도 상 → 하로 완전히 해소됨.
✅ 표에서 읽을 수 있는 것

Before에서 M-02가 심각도 상(上)으로 즉시 개선 대상임이 한눈에 보인다. After에서 두 미션 모두 심각도가 하(下)로 내려갔다.

⚠ 표만으로는 알 수 없는 것

M-01의 소요 시간이 15.1s → 31.3s로 두 배 늘었다. 성공률은 올랐지만 왜 더 오래 걸렸는지는 표만으로 알 수 없다. 다음 단계인 Behavior Mapping이 필요하다.

💡 단일 지표를 믿지 말 것

성공률만 보면 M-01은 Before에서도 85.7%로 "괜찮아 보인다". 그러나 Misclick 79.6%를 함께 보면 전혀 다른 이야기가 된다. 항상 여러 지표를 함께 읽자.

행동 데이터와 태도 점수를 나란히 놓기

실제 행동 수치(성공률, Misclick)와 사용자가 느낀 만족도를 같은 기준으로 나란히 놓고 비교하는 방법이다. 두 데이터가 일치하지 않는 곳에서 숫자만 봐서는 절대 알 수 없는 문제가 드러난다.

이럴 때 사용해요
성공률은 높은데 사용자 반응이 차갑게 느껴질 때.
개선 후 숫자가 좋아졌는데 팀원들이 실감을 못 할 때.
어떤 부분이 가장 불만족스러운지 우선순위를 잡고 싶을 때.
행동 데이터 태도 데이터
4가지 조합 패턴
← 태도 낮음 태도 높음 →
성공률 高 + 태도 高
잘 작동하는 영역
사용자가 성공하고 만족했다. 유지한다.
→ 현상 유지. 다른 기능 개선의 기준점으로 활용.
성공률 低 + 태도 高
접근성 문제
못 찾았지만 경험 자체는 나쁘지 않았다. 기능보다 경로 문제.
→ 탐색 경로와 정보 구조(IA) 개선 집중.
성공률 高 + 태도 低 ⚡
숨겨진 마찰
어떻게든 해냈지만 경험이 불쾌했다. 숫자만 보면 절대 모른다.
→ 정성 조사 필수. 인터뷰로 마찰 원인 파악.
성공률 低 + 태도 低
명백한 문제
실패했고 만족도도 낮다. 즉시 개선 필요.
→ 즉시 개선. 카카오뱅크 Before M-02가 이 케이스.
